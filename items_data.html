RDF data related to the items

A note on the process: we were only asked to create a csv file for each item but after researching some processes for transforming and visualising RDF data I decided to go a little further to explore interoperability and accessibility.

I began with manually creating the csv files, then I turned them into html tables (below) using a csv to html converter. At this point I had already started creating the final RDF file the project asked for, and so I also turned the items data into their own RDF files, using Turtle syntax, which I then fed into a converter to generate JSON-LD versions. These were in turn fed into the JSON-LD playground to create compact versions. All three versions (validated for any errors) of each item are available directly from this page and the GitHub repository of the project. I had to stop there due to time constraints but I would like to further explore how to use these different formats to make the data available, as well as to play with it visually and plug it into the wider semantic web.

As per the requirements, each item is described using natural language for subject and object and the specific predicate from the chosen vocabularies. Each item table is also a URI for itself formatted as domain/items_data.html#NAME with the name located in the id attribute of the HTML title tag. As such accessing https://laurentfintoni.github.io/jayinlodee/items_data.html#Welcome2Detroit directly will lead you to the relevant location on this page.
